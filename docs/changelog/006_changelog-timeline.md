# 变更日志与时间线增强

> **日期**: 2026-03-29  
> **分支**: feature/changelog-timeline

## 🎉 新增功能

- **变更日志展示页**: 在「系统设置」目录下新增了「变更日志」菜单，提供全站可视化的系统迭代记录展示。
- **时间线布局**: 采用双栏布局设计，左侧目录树，右侧时间线，方便快捷总览与查阅详细历史记录。
- **全局搜索能力**: 支持对变更标题、日期、分支、内容章节等全文字段的实时检索匹配，自动高亮展开对应章节。
- **动态解析服务**: 后端自动读取并解析 `docs/changelog/` 目录下的 Markdown 日志，按变更序号动态生成渲染数据。

## 🔧 重构与优化

- **语义化配色**: 依据变更日志的章节标题（如功能、重构、修复、迁移），为时间线节点自适应生成并匹配相应的视觉主题色与图标，提升信息获取效率。
- **深色模式适配**: 时间线、目录树和搜索框已完全适配 UI Pro Max 设计规范下的深浅主题色，卡片边框色彩柔和克制。
- **沉浸式交互**: 实现交叉观察器（IntersectionObserver），实现滚动正文时左侧目录高亮随动效果。

## 📝 文档变更

- 更新所有旧版变更日志文件，增加 `00N_` 数字前缀用于规范按顺序排列系统演进历程。
- 添加 `006_changelog-timeline.md` 记录本次特性改动。
