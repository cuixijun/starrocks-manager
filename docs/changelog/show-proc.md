# 高级诊断 (SHOW PROC) 功能及 UI 规范重构

**日期**: 2026-03-29
**模块**: 高级诊断 (SHOW PROC)，合并诊断 (Compaction Score)

## 核心功能新增

*   **SHOW PROC 目录视图**:
    *   按照 5 个大类（集群基础设施、数据与元数据、任务与作业、查询与监控、资源管理）结构化展示了 StarRocks 系统支持的 18 个 `SHOW PROC` 监控项。
*   **深层路径下钻架构**:
    *   实现了严格的层级下钻能力（如 `dbs -> DBID -> TableID -> partitions`），能够根据官方层级规范穿透到系统底层查看分片级别的元数据。
*   **可视化路径导航**:
    *   增加了支持层级返回的交互式面包屑导航条，清晰展示当前所处的路径深度 `SHOW PROC '/dbs/...'`。
*   **后端安全查询层**:
    *   设计了基于静态白名单的 `SHOW PROC` 数据查询 API，彻底杜绝下钻路径带来的 SQL 注入风险。
*   **智能列数据格式化**:
    *   自动识别并美化常见数据类型：IP（Mono 字体）、系统状态（带 Badge 徽标的在线/离线）、水位百分比（进度条）、系统角色（LEADER 专属高亮）。

## UI Pro Max 规范对齐

*   **卡片化头部设计**:
    *   所有高级诊断详情页面的标题栏区域重构为带有拟态阴影与左侧渐变徽章的卡片容器。
*   **工具栏标准化 (`.table-toolbar`)**:
    *   统一了所有页面的高级列表工具栏结构：左侧为搜索输入框（240px，带前置图标），中部为筛选器下拉（160px），右侧收敛为操作按钮区。
    *   搜索功能全局添加了“当前结果 x / n 行”的显式指示。
*   **下拉组件替换**:
    *   淘汰了「物化视图管理」与「合并诊断」页面生硬的原生 `<select>`。
    *   全面换用 `SearchableSelect` 组件，不仅支持快速拼音/模糊检索，同时 UI 弹出层（带有入场动效、边框反馈）也与整个 Manager 项目对齐。

## 合并诊断 (Compaction Score) 优化

*   **命名规范化**:
    *   将晦涩的“Compaction Score”菜单全局更名为理解度更高的“合并诊断”。
*   **性能减负**:
    *   移除了该页面顶层厚重且低频使用的全局全量监控统计卡片（`SUM(CASE WHEN...)` 等耗时操作）。
    *   数据库筛选器重构为实时请求 `SHOW DATABASES` 过滤系统库后按字母排序生成。
*   **表格防抖动**:
    *   分页刷新（如页面切换或点击刷新按钮时）增加了覆盖于表格之上的透明 Loading 蒙层（`.loading-overlay`），彻底消除了原先重新拉取数据导致整个表格高度塌陷（Jitter）的恶劣体验。
